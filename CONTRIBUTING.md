# Contributing

Спасибо за интерес к Design Buddy.

## Перед изменениями

- Прочитайте `docs/MVP.md`: это основной источник продуктовых и технических требований.
- Сохраняйте простую архитектуру проекта и текущий one-shot запуск агента через `codex exec`.
- Не добавляйте прямые вызовы API LLM-провайдера, постоянные agent-сессии, RAG, Redis, Celery, FastAPI или multi-agent framework без отдельного обсуждения.

## Локальная проверка

Перед pull request запустите:

```bash
ruff check .
pytest
```

Тесты должны использовать fake agent runner. Не запускайте реальный агент и сетевые вызовы из тестов.

## Секреты

Не коммитьте `.env`, Telegram-токены, личные Telegram ID, базы SQLite с реальными данными и другие локальные секреты. Для примеров используйте `.env.example`.
