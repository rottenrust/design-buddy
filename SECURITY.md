# Security

## Секреты и локальные данные

Не публикуйте Telegram bot token, реальные Telegram ID, `.env`, локальные SQLite-базы с пользовательскими данными и любые другие секреты. `.env.example` должен содержать только пустые или шаблонные значения.

## Сообщение об уязвимости

Если вы нашли уязвимость, создайте приватный security advisory, если платформа репозитория это поддерживает. Если такой возможности нет, свяжитесь с сопровождающими проекта через issues без публикации токенов, персональных данных и эксплуатационных деталей.
