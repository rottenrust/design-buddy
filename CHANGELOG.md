# Changelog

Все заметные изменения Design Buddy документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), версии следуют [Semantic Versioning](https://semver.org/lang/ru/).

## [0.2.0] — 2026-07-05

Первая тегированная публичная версия Design Buddy как персонального Telegram-наставника для начинающего UX/UI-дизайнера.

### Добавлено

- Первичный последовательный опрос дизайнера.
- Постоянный профиль по 12 основным UX-навыкам.
- Каталог из 12 практических заданий в `data/tasks.yaml`.
- Пошаговый сценарий выполнения активного задания.
- Объяснения, примеры и микро-задания для текущего шага.
- Проверка результата по заранее заданной рубрике.
- Сохранение evidence и обновление профиля навыков.
- Долговременная персональная память.
- Rolling summary и ограниченный контекст последних сообщений.
- Команды просмотра заданий, профиля и текущего прогресса.

### Изменено

- Design Buddy превращён из свободного дизайн-чата в управляемого UX-наставника.
- Состояние пользователя, активное задание и прогресс контролируются backend, а не agent-сессией.
- Каждый запрос запускает отдельный процесс `codex exec` без постоянной или resume-сессии.
- README обновлён для публичного репозитория и релиза v0.2.0.

### Техническая основа

- Python 3.12.
- aiogram 3.
- SQLAlchemy 2 и SQLite.
- Pydantic и Pydantic Settings.
- Codex CLI.
- pytest и ruff.

### Ограничения

- Single-user режим.
- Long polling вместо webhook.
- Нет веб-интерфейса, RAG, vector store, PostgreSQL и постоянных agent-сессий.
